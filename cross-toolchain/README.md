

## 1: 打印样本列表(可选).
```
$ ./list-samples.sh
```

## 2: 参数配置, 按需要修改.
```
$ ./configure.sh [sample]
```
* 根据需要修改下面的配置:
* 当使用wget作为下载工具出错证书过期或无效时, 可以通过添加--no-check-certificate忽略.
* 内核, 建议使用稳定版本.
* binutils, 建议使用2.40(或特定版本), 否则较早的汇编代码可能会因为安全检查无法编译通过.
* GCC扩展配置, 建议启用-fcommon参数, 因为从GCC10(包含)开始默认使用-fno-common配置, 禁用后编译较早的代码可能会出现各种兼容性错误.
* GCC扩展配置, 建议启用--with-gcc-major-version-only参数. 因为从GCC7(包含)开始各平台发行版都默认启用此配置.
* 优化LOOP, 建议使用默认值, 除非明确知道其作用.
* LTO, 建议使用默认值, 除非明确知道其作用.
* PIE, 建议不要设置为默认值, 除非明确知道其作用.
* 优化libs大小, 建议禁用, 除非明确知道其作用, 否则宏_GLIBCXX_USE_C99_MATH将会默认禁用(因为GCC某些BUG,浮点支持检测会失败).
* libmudflap, 建议禁用, 除非明确知道其作用.
* libgomp, 建议启用, 否则一些需要并行计算的项目将无法加速.
* libssp, 建议禁用, 除非明确知道其作用, 否则一些安全敏感程序无法编译通过.
* libquadmath, 建议禁用, 除非明确知道其作用.
* libsanitizer, 建议禁用, 除非明确知道其作用.
* libmpx, 建议禁用, 除非明确知道其作用.
* TLS, 建议使用默认值, 除非明确知道其作用.
* sjlj, 建议使用默认值, 除非明确知道其作用.
* 128位长双精度支持, 建议禁用, 除非明确知道其作用, 否则宏_GLIBCXX_USE_C99_MATH将会默认禁用(因为GCC某些BUG,浮点支持检测会失败).

## 3: 构建.
```
$ ./build.sh
```
* 无论是源码下载还是编译构建, 所花费的时间都超出预期,耐心等待即可.
* 如果构建过程提前中断,查看工作目录下构建日志(build.log), 根据日志提示从第2步开始修复配置, 直到构建完成.
